# Interactive Quiz Kit

[![NPM version](https://img.shields.io/npm/v/@thanh01pmt/interactive-quiz-kit.svg)](https://www.npmjs.com/package/@thanh01pmt/interactive-quiz-kit)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

**Interactive Quiz Kit** is a comprehensive, modular TypeScript library for React, designed to effortlessly create, manage, and deploy interactive quizzes and **personalized learning applications**. It provides a robust headless core, a rich suite of reusable React UI components, and powerful AI-driven features for content creation, intelligent analysis, and knowledge reinforcement.

The library is architected for maximum flexibility, featuring a powerful authoring tool, AI content generation (using Google's Genkit and Gemini), and SCORM packaging for seamless LMS integration.

## ✨ Features

The library is packed with features designed to cover the entire learning and assessment lifecycle.

### 🧠 Core & Authoring

- **Robust Headless Engine**: A `QuizEngine` that handles state management, navigation, automatic grading, and time tracking, usable in any JavaScript environment.
- **Rich Question Support**: Includes 12+ question types, from Multiple Choice to complex ones like Blockly/Scratch Programming and Hotspot.
- **Powerful Authoring Tool**: A complete `<QuizAuthoringTool>` component for creating, editing, and managing quizzes with an intuitive UI.
- **Flexible Content Import**: Import questions directly from JSON or TSV files to quickly populate your quizzes.

### 🤖 AI-Powered Capabilities

- **Intelligent Content Generation**:
  - Generate individual questions based on topic, context, and difficulty.
  - Generate entire, well-structured quizzes using a sophisticated two-stage planning and generation process.
- **Intelligent Document Analysis**: Upload a document (.txt, .pdf, .docx), and the AI will automatically assess its relevance to your existing curriculum, map it to specific Learning Objectives, or recommend "Freestyle Mode" for unrelated content.
- **AI-Generated Knowledge Cards (Cheatsheet)**: Automatically analyze imported curriculum data to identify key concepts and generate a full set of detailed knowledge cards (flashcards), complete with definitions and examples. The generation process is resumable and runs in the background.
- **AI Tutor for Practice & Review**:
  - **Personalized Suggestions**: An AI tutor analyzes user performance to suggest a custom practice plan, focusing on weak areas and related new topics.
  - **Detailed Remediation**: After each quiz, generate an in-depth, AI-written review that explains concepts, clarifies misconceptions, and provides a summary for reinforcement.
  - **Context-Aware Chatbot**: Engage in a conversation with an AI Tutor that understands your learning history, goals, and available topics to provide personalized answers.

### 📊 Personalized Learning & Analytics

- **Personal Practice Dashboard**: A high-level `<PersonalPracticeDashboard>` component that serves as a rich, engaging, and customizable learning hub for students.
- **Interactive Knowledge Cards**: A `<Cheatsheet>` widget that allows users to search, browse, and review the AI-generated knowledge cards.
- **Performance Insights**: Visualize progress over time and analyze strengths/weaknesses by topic with built-in charts and a GitHub-style activity calendar.
- **Detailed & Interactive History**: Review every practice session. Click on any past session to retake the quiz, view detailed results, or read the AI-generated review.
- **Dedicated Quiz Lists**:
  - **Generated Quizzes**: A filterable list of all unique, curriculum-aligned quizzes generated by the AI for Easy retakes.
  - **Freestyle Quizzes**: A separate list for quizzes generated from uploaded documents that don't map to the main curriculum.
- **Gamification Engine**: Motivate learners by tracking daily practice streaks, progress towards weekly goals, and unlocking a wide range of achievements.
- **LMS & Webhook Integration**:
  - Package quizzes as SCORM 1.2 compliant ZIP files.
  - Send detailed quiz results to a specified webhook URL for external data processing.

## 📚 Usage & Architecture

This library is modular, providing multiple entry points for optimal performance and tree-shaking.

- **`@thanh01pmt/interactive-quiz-kit`**: The core **headless** library (types, services). Use this for backend or non-React logic.
- **`@thanh01pmt/interactive-quiz-kit/react-ui`**: The full suite of **React UI components**, including the Dashboard and Authoring Tool.
- **`@thanh01pmt/interactive-quiz-kit/player`**: A lightweight bundle for the **quiz player**. Ideal for SCORM packages.
- **`@thanh01pmt/interactive-quiz-kit/ai`**: All **AI-related** generation and analysis functions.

### 1. Quick Start: Building a Complete Practice App

The fastest way to get a full-featured, personalized learning application is by using the high-level `<PersonalPracticeDashboard>` component.

#### Example: Displaying the Practice Dashboard

```tsx
import React from "react";
// Import from the main UI entry point
import { PersonalPracticeDashboard } from "@thanh01pmt/interactive-quiz-kit/react-ui";

const MyPracticeDashboardPage = () => {
  // The dashboard component handles its own state and data fetching
  // from localStorage using the library's headless services.
  return (
    <div className="container p-8">
      <PersonalPracticeDashboard />
    </div>
  );
};
```

### 2. Using Specific UI Components

You can also build custom interfaces by composing specific components from the library.

#### Example: Displaying a Standalone Quiz Player

```tsx
import React from "react";
import { QuizPlayer } from "@thanh01pmt/interactive-quiz-kit/react-ui";
import type { QuizConfig, QuizResultType } from "@thanh01pmt/interactive-quiz-kit";

const myQuiz: QuizConfig = { /* ... your quiz configuration ... */ };

const MyQuizPage = () => {
  const handleQuizComplete = (result: QuizResultType) => {
    console.log("Quiz Complete!", result);
  };

  return (
    <div className="container">
      <QuizPlayer quizConfig={myQuiz} onQuizComplete={handleQuizComplete} />
    </div>
  );
};
```

#### Example: Integrating the Authoring Tool

#### Example: Intelligent Document Analysis Pipeline

This example shows how to use headless services and AI flows to create the "Generate Quiz from Document" feature.

```typescript
// Import headless services and AI flows
import { TopicDataService } from "@thanh01pmt/interactive-quiz-kit";
import { assessAndMapDocument } from "@thanh01pmt/interactive-quiz-kit/ai";
import type { LearningObjective } from "@thanh01pmt/interactive-quiz-kit";

async function analyzeDocument(documentText: string, apiKey: string) {
  try {
    // 1. Gather curriculum data using a headless service
    const allLearningObjectives: LearningObjective[] = TopicDataService.getData();

    // 2. Call the AI triage flow to assess and map the document
    const analysis = await assessAndMapDocument({
      language: "Vietnamese",
      documentContent: documentText,
      learningObjectives: allLearningObjectives,
    }, apiKey);

    // 3. Use the analysis for backend logic or a custom UI
    console.log(`Document Relevance Score: ${analysis.relevanceScore}`);
    if (analysis.isFreestyleRecommended) {
      console.log("Recommendation: Proceed with Freestyle Mode.");
      // Trigger freestyle quiz generation...
    } else {
      console.log("Recommendation: Integrate into curriculum.");
      console.log("Mapped Learning Objectives:", analysis.mappedLOs);
      // Trigger structured quiz generation using the mapped LOs...
    }

  } catch (error) {
    console.error("Error analyzing document:", error);
  }
}
```

## 💡 Architecture Philosophy

The library is built on a core philosophy of modularity and separation of concerns, making it powerful and flexible.

- **Headless First:** All core business logic—quiz engine, data management, AI flows, and analytics—is implemented in pure, environment-agnostic TypeScript services. This ensures the "brain" of the library can be used anywhere, from a Node.js backend to a mobile app.
- **Composable UI Layer:** The React UI (`react-ui`) is a consumer of the headless core. It provides a comprehensive, ready-to-use implementation but is entirely optional. You can build your own interface in any framework by interacting with the headless services directly.
- **Stateful on the Client:** The application is designed to be fully functional on the client-side, using `localStorage` for data persistence. This allows for a fast, serverless-first user experience.

## ⚠️ Known Issues & Limitations

- **Asset Management for Programming Questions**: Using `Blockly` or `Scratch` question types requires manually copying their respective asset files (`js`, `css`, `media`) from their `node_modules` packages into your project's `public` folder for the UI components to render correctly.
- **SCORM Packaging**: The `exportQuizAsSCORMZip` function is a browser-based utility that fetches assets from your `public` folder to create a complete package. Ensure your built library files (`player.js`, `styles.css`) are present in the expected public paths.
- **AI Generation Limits**: AI generation is not yet supported for highly visual or structurally complex question types like `Drag and Drop`, `Hotspot`, `Blockly`, or `Scratch` during the full quiz generation process. They must be authored manually.
- **Document Parsing**: The "Generate Quiz from Document" feature requires client-side libraries like `pdfjs-dist` (for PDF) and `mammoth` (for DOCX) to be installed in your project to handle file content extraction.
